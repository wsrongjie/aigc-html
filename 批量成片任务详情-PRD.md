# 批量成片 · 任务详情 PRD

## 一、页面结构

### 父表（任务级）

| 列 | 数据源 | 说明 |
|---|---|---|
| 任务名 | `taskName` | 带展开/收起按钮，点击展开子表 |
| 可见范围 | `visibility` | 枚举：`仅自己` / `团队可见` |
| 视频数量 | 子任务计数 | 数值 |
| 总消耗积分 | 子任务 `cost` 求和 | 数值 + 🪙 |
| 任务创建时间 | 最早子任务的 `createdAt` | 格式 `yyyy-MM-dd HH:mm:ss` |
| 任务状态 | 聚合计算（见第三节） | 带颜色区分 |
| 任务进度 | 完成数 / (非草稿总数) × 100 | 百分比 |
| 任务完成时间 | 最晚子任务的 `completedAt` | 格式同上，无则显示"—" |
| 操作 | 按钮组（见第五节） | 按状态控制启禁 |

### 子表（单条视频级）

| 列 | 数据源 | 说明 |
|---|---|---|
| 首帧 | 角色 emoji + 背景渐变色 | hover 显示角色 + 背景名称 tooltip |
| 台词 | `script` | hover 全文 tooltip，超 25 字截断显示"…" |
| 视频时长 | `duration` | 单位秒，如 `8s` |
| 消耗积分 | `cost` | 数值 + 🪙 |
| 视频状态 | `status`（见第四节） | 带颜色和图标 |
| 生成进度 | `progress` | 进度条 + 百分比，排队中/失败显示"—" |
| 开始生成时间 | `startedAt` | 格式化 |
| 生成完成时间 | `completedAt` | 格式化 |
| 生成耗时 | `startedAt` → `completedAt` | 自动计算，如"2分30秒" |
| 操作 | 按钮组（见第五节） | 按状态控制启禁 |

## 三、父级状态聚合（7种）

优先级从高到低匹配：

| 优先级 | 匹配条件 | 显示状态 | 颜色 |
|---|---|---|---|
| 1 | 全部子任务为 `draft` | 草稿 | `#888` |
| 2 | 任意子任务 `generating` 或 `wmStatus=processing` | 生成中 | `#0066ff` |
| 3 | 全部 `queued`，无其他状态 | 排队中 | `#f59e0b` |
| 4 | 全部 `failed` | 已失败 | `#ef4444` |
| 5 | 全部 `cancelled` | 已取消 | `#999` |
| 6 | 全部 `completed`（无 failed/generating/queued/cancelled） | 已完成 | `#059669` |
| 7 | `completed > 0` 且 (`failed > 0` 或 `cancelled > 0`)，且无 generating/queued | 部分成功 | `#f59e0b` |
| 兜底 | 其他混合态 | 组合文本（如"2完成,1生成中"） | `#333` |

## 四、子任务状态机

```
草稿 → 排队中 → 生成中 → 已完成 → 水印合成中 → 已完成(wm done)
                     ↘ 取消                       ↗ 失败
                       → 已取消        重试 →
```

| 状态 | 枚举值 | 判定条件 | 颜色 | 图标 |
|---|---|---|---|---|
| 草稿 | `draft` | `status=draft` | `#888` | 📝 |
| 排队中 | `queued` | `status=queued` | `#f59e0b` | ⏳ |
| 生成中 | `generating` | `status=generating` | `#0066ff` | 🔄 |
| 已完成 | `completed` | `status=completed`，`wmStatus!=processing` | `#059669` | ✅ |
| 水印合成中 | `completed` | `status=completed`，`wmStatus=processing` | `#f59e0b` | ⏳ |
| 已失败 | `failed` | `status=failed` | `#ef4444` | ❌ |
| 已取消 | `cancelled` | `status=cancelled` | `#ef4444` | 🚫 |

## 六、统一取消弹窗交互

父子两级取消按钮共用一个弹窗。

### 弹窗结构

```
┌──────────────────────────┐
│          ⚠️              │
│       确认取消            │
│                          │
│  确认取消「任务名」中      │
│  N 条视频的生成？          │
│  已完成的视频不受影响。    │
│                          │
│   [再想想]   [确认取消]   │
└──────────────────────────┘
```

### 父级取消流程

1. 收集该批次下所有 `queued` + `generating` 的子任务
2. 无此类子任务 → toast 提示"该任务没有正在排队或生成中的视频"，不弹窗
3. 有 → 弹窗，描述：`确认取消「{任务名}」中 N 条视频的生成？已完成的视频不受影响。`
4. 确认 → 选中子任务 `status = cancelled`，`progress = 0`，清除 `error`

### 子表取消流程

1. 单条视频，确认其状态为 `queued` 或 `generating`
2. 弹窗，描述：`确认取消该条视频的生成？`
3. 确认 → 同父级，状态改为 `cancelled`

### 二次确认关闭

- 点击"再想想"或弹窗遮罩层 → 关闭弹窗，不做任何操作
- 确认后刷新列表
